# Current State & Future

Everything on this blog — the writing, the experiments behind the writing, the hardware that runs the experiments — is funded out of pocket. This page exists to be transparent about what that means and what could change.

## What I invest

I'm a systems architect. That's where I'm productive and where I belong. But the infrastructure gap described in this blog won't document itself, and the experiments won't run themselves. So I fund both — personally, on my own time.

**Hardware.** I have sufficient modern consumer and prosumer hardware in hand — enough to run and test local models up to medium scale, benchmark inference patterns, validate the architecture described in these articles against real workloads.

**Subscriptions and API credits.** Claude, Google One AI, Cursor, Windsurf, Qoder, Vertex AI and a rotating set of others depending on what's being tested. Hundreds of dollars per month in subscriptions and token costs. Every experiment behind every article — the RAG evaluations, the distillation runs, the agent session analysis — burns real credits on real providers and electricity of my household.

**Time.** Evenings, weekends, vacation days. No employer hours, no employer hardware, no employer data.

This is enough to research, prototype, and write. It is not enough to build production-grade open-source tooling at the scale the architecture implies — and that's the honest constraint.

## What would help

I'm a software and systems architect — that's where I'm most comfortable and most productive. I'm not trying to become a blogger. But the research, the experiments, and the writing take real resources, and doing them better takes more than I can sustain alone.

**Offset the running costs.** A small recurring contribution that covers API credits and subscriptions. At enough supporters, the difference is running the next experiment without watching the meter.

**Subscription or token grants.** If you work at a company with API credits, model access, or cloud compute to spare — those translate directly into experiments and articles. No cash changes hands.

**Hardware access.** Modern datacenter GPUs (H100, MI300, B200 class) for inference experiments beyond what consumer hardware can do. FPGA development boards beyond the MiSTer ecosystem. Either short-term loans tied to a specific article or permanent donations — both useful in different ways.

**Collaboration or sponsorship.** For organizations that want a deeper relationship: joint experiments, sponsored deep-dives, named acknowledgments. This requires structure — LLC, contracts, defined deliverables — and only makes sense if demand justifies it.

**Hiring me.** The fastest path to more articles is for someone to pay me to think about this full-time. I'm open to that conversation.

## How this scales up

Right now there is no LLC, no donations page, no Patreon, no sponsorship structure. The reason is simple: setting any of that up takes meaningful effort, and I'd rather not do it if there's no demand.

So the current state is: write the articles, see who reads them, see what people say. If there's clear interest, the infrastructure follows — and this page gets updated with real links instead of descriptions.

The [How to Help](./04-how-to-help.md) page has a quick survey to gauge interest. If you've read this far, please take 30 seconds to fill it out.
